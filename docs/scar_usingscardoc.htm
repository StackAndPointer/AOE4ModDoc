使用ScarDoc

ScarDoc可用于添加自动文档化的C++和Lua函数。  要让函数显示在此网页中，您只需在函数上方添加一些特殊注释即可。  您可以在函数上使用多种不同的标签；有些是必需的，而其他则是可选的。

使用ScarDoc为Lua函数编写文档时，必须为函数的参数和返回值提供类型信息。  这样ScarDoc才能知道需要向函数传递哪些类型。  用于指定这些信息的标签是@result和@args。

示例 1.1：使用ScarDoc注册Lua函数
--? @shortdesc 设置玩家的种族。
--? @extdesc
--? 种族名称必须是属性编辑器中的有效种族名称。\n\n
--? 如果raceName不存在，此函数将返回false。\n\n
--? 此注释将位于新段落中。
--? @result Void
--? @args PlayerID playerID, String raceName
--? @refs PlayerRaces.txt, SomeOtherDoc.doc
function Player_SetRace( playerID, raceName )
    --
end

为C++函数编写文档则更为直接，因为类型信息已在C++语言中指定。  由于C++类型名称可能看起来非常复杂，ScarDoc会将C++类型名称转换为"设计者友好"的类型名称。  这样，如果Lua绑定函数接受"const Math::Vector3f  *"作为参数，编写脚本的人员无需担心其所有含义。  在这种情况下，它将显示为更友好的名称，如"PositionRef"。  要查看所有类型及其转换列表，请转到>>待完成<<。 

示例 1.2：使用ScarDoc注册C++函数
//? @shortdesc 设置玩家的种族。
//? @extdesc
//? 种族名称必须是属性编辑器中的有效种族名称。\n\n
//? 如果raceName不存在，此函数将返回false。\n\n
//? 此注释将位于新段落中。
//? @refs PlayerRaces.txt, SomeOtherDoc.doc
static bool Player_SetRace( Player * playerID, const char * raceName )
{
}

函数还可以分类到命名的子组中。  这些是您在函数列表中看到的子组。要设置当前组，请使用@group标签。  一旦设置了组，文件中该标签下方的所有函数都将添加到此组中。  如果找到另一个@group标签，这将更改当前组，函数现在将添加到该组中。  如果不指定组，函数将添加到"Various"（杂项）组。

示例 1.3：使用@group标签
/* 在此示例中，"Group1"将包含Function1和Function2，"Group2"将包含Function3和Function4。  Function0将被添加到"Various" */

//? @shortdesc Function0的描述。
static void Function0( ){}


//? @group Group1

//? @shortdesc Function1的描述。
static void Function1( ){}

//? @shortdesc Function2的描述。
static void Function2( ){}


//? @group Group2

//? @shortdesc Function3的描述。
static void Function3( ){}

//? @shortdesc Function4的描述。
static void Function4( ){}

 

标签

以下是所有可能的ScarDoc标签列表。

  • @shortdesc（必需）
    • 函数的一句话描述。  此描述将显示在WSciTE API中。
  • @extdesc（可选）
    • 函数的可选详细描述。  如果在描述中放入"\n"，生成的HTML将包含<BR>标签。此描述不会显示在WSciTE API中。
  • @refs（可选）
    • 添加指向外部文档的链接。  文件路径必须相对于scardoc/html/references文件夹。   如果有多个文件名，请用逗号分隔。
  • @result（必需，仅Lua）
    • 用于指定函数的返回值类型。  此返回值类型必须是有效的ScarDoc类型。
  • @args（必需，仅Lua）
    • 用于指定函数参数的类型。  如果此列表中的参数数量与函数中的参数数量不匹配，ScarDoc将在其日志文件中显示警告。
  • @group（可选）
    • 此标签不必属于函数的一部分。这是当前文件的全局标签，用于设置当前组类型。  如果未指定此标签，函数将被添加到"Various"组。
